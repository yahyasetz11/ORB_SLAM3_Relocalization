# Session Summary — 2026-05-28

## 1. Replaced Hamming-only filter with RANSAC + LM pipeline in `processFrame()`

**File:** `src/orb_slam3_relocalization/src/relocalization.cpp`

Old pipeline:
- matchWithKeyFrame → Hamming threshold filter → solvePnPWeighted (LM, uniform weights)

New pipeline:
1. `matchWithKeyFrame()` → correspondences + Hamming distances
2. **Loose Hamming pre-filter** — discard obvious mismatches (`hammingDist < HammingPreFilter`, default 100)
3. **RANSAC** (50 iterations) — sample 4 points, solve with `cv::SOLVEPNP_P3P`, score all pre-filtered matches using 3D camera-space distance:
   - `X_c = R·X_w + t`, `X_c' = Zc·[(u−cx)/fx, (v−cy)/fy, 1]`
   - Inlier if `||X_c − X_c'|| < RansacInlierThreshold3D` (default 0.10 m)
   - Keep pose with most inliers
4. **LM pass 1** — `solvePnPWeighted()` (uniform weights) on all RANSAC inliers, warm-started from RANSAC best pose
5. **LM pass 2** — `solvePnPWeighted()` (uniform weights) on the strict 8px reprojection-error subset from LM pass 1 → this is the final Std PnP result

The double LM pass ensures the Std PnP output uses only geometrically strict correspondences, making it equivalent to the node-layer WPnP when `background_weight = 1.0`.

## 2. Replaced confidence-score ranking with geometric ranking

**File:** `src/orb_slam3_relocalization/src/relocalization.cpp`

Old ranking formula (0–100):
```
confidence = inlierRatio×100 + bowScore×1200 + min(50, inliers)×0.8
```

New ranking (best candidate keyframe selection):
```
rankScore = lm2.numInliers / max(lm2.meanReprojectionError, 0.01px)
```
Higher inlier count and lower reprojection error both improve the score. BoW score no longer influences candidate selection (it still gates which keyframes are considered via `detectRelocalizationCandidates`).

`result.confidence` is kept as `inlierRatio × 100` for the node's display color thresholds.
